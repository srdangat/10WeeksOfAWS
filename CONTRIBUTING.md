# Contributing to #10WeeksOfAWS

Thanks for being part of the challenge. This repo works a little differently from a typical open-source project, so here's how to get the most out of it.

## Taking part as a learner

The main way you participate is by completing each week's hands-on task and
submitting your proof of work through a pull request.

1. Star and fork this repo so you have the weekly plan.
2. Do the week's labs in your own AWS account.
3. Add your notes, screenshots, diagrams, policies, or code under the weekly
   submission folder.
4. Open one pull request per week.
5. Post weekly on LinkedIn, X, or GitHub with the tags:

   ```
   #10WeeksOfAWS   #CloudAdhar   #TrainWithShubham
   ```

6. Tag [@TrainWithShubham](https://youtube.com/@TrainWithShubham) and [@cloudadhar](https://youtube.com/@cloudadhar) so we can follow along and reshare.

Use [SUBMISSION_GUIDE.md](SUBMISSION_GUIDE.md) for the exact folder structure,
PR title format, and weekly checklist.

## Improving the challenge content

Spotted a typo, a broken link, an outdated AWS step, or want to suggest a better lab? Those contributions are very welcome:

1. Open an issue describing the problem or idea.
2. For fixes, fork, branch, and open a PR against `main` with a clear description.
3. Keep changes concise and consistent with the existing week README format: Overview, Theory & Concepts, Hands-on Labs, Carry-Forward, Exam Domain Mapping, Learn in Public.
4. Keep the tone professional and beginner-friendly.

## Ideas we especially value

- Clearer lab steps or cleanup instructions
- Additional sample exam questions with explanations
- Architecture diagrams for a week's topic
- Anything that prevents unexpected AWS billing

## Code of Conduct

By taking part you agree to follow our [Code of Conduct](CODE_OF_CONDUCT.md). Be kind, be helpful, and celebrate each other's progress.
